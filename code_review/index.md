## 为什么要 review 代码

- 提高代码质量。避免坏味道的代码。
- 形成团队统一的高质量标准。团队成员会有意识地关注代码质量，从而形成越来越高的事实上的质量标准。 
- 个人快速成长。双向交流的过程。

## 如何发起 review

- 遵循 commit 规范。遵循 Angular commit 规范。
- 自己 review 一遍。
- 提 MR。
    - 单一
    - 短小
    - 详细。描述背景和动机。

## 如何 review 代码

**简单快速 review**

询问 author 是否实际环境中验证过。

**常规 review**

- 代码可读性。
- 代码健壮性。
- 代码扩展性。
